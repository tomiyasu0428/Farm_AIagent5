# 農業AIエージェント 開発タスクリスト

## フェーズ1: 基盤構築と読み取り機能

- [ ] **環境構築**: Google Cloud, MongoDB, LINE Developersの基本的な設定を行う。
- [ ] **バックエンド基盤**: FastAPIを使用して、LINEからのWebhookを受け付けるエンドポイントを作成する。
- [ ] **LangGraph初期設定**: グラフの基本となる`State`（状態）を定義し、会話の永続化のために`MongoDBSaver`を設定する。
- [ ] **基本エージェント作成**: 会話の司令塔となる`Supervisor`（ルーター役）と、データの読み取りを担当する`ReadAgent`の雛形を作成する。
- [ ] **基本フロー実装**: LINEでの簡単な質問に対し、`Supervisor`が`ReadAgent`に処理を渡し、データベースから取得した情報をLINEに返すまでの一連の流れを完成させる。

## フェーズ2: 書き込み機能とUIの基本

- [ ] **書き込み機能実装**: 作業記録の登録など、データベースに情報を書き込むための`WriteAgent`と、それに対応するツールを作成し、グラフに追加する。
- [ ] **安全機構の実装**: データベースの重要な情報を更新する前に、LINEを通じてユーザーに「実行しますか？」と確認を求めるフロー（Human-in-the-Loop）を実装する。
- [ ] **LIFF基本開発**: その日のタスク一覧や、簡単な圃場情報を表示するための、基本的なLIFF（LINE Front-end Framework）ダッシュボードを作成する。

## フェーズ3: AI機能の高度化

- [ ] **推薦機能実装**: 天候データや過去の作業履歴を分析し、「次の作業」や「最適な農薬」を提案する`RecommendationAgent`と、そのための分析ツールを開発する。
- [ ] **CodeActパラダイム導入**: `eval`などの危険な方法は避け、安全なサンドボックス環境を準備した上で、簡単なデータベースクエリをAIに動的に生成・実行させる自己修正サイクルを試作する。
- [ ] **LIFF機能拡張**: 過去の作業履歴をグラフで表示したり、カレンダー形式で作業計画を可視化するなど、より高度な機能を追加する。

## フェーズ4: 自律システムの実現

- [ ] **非同期処理基盤構築**: Cloud Pub/Subなどを利用し、時間のかかる処理や定期実行タスクをバックグラウンドで非同期に実行する仕組みを構築する。
- [ ] **プロアクティブ通知機能実装**: `NotificationAgent`と定時実行ジョブ（Cron）を組み合わせ、「明日は雨予報です」といった能動的なアラートをLINEに送信する機能を実装する。
- [ ] **観測可能性とセキュリティ**: LangSmithを導入してエージェントの動作をトレース可能にし、本番運用に向けたセキュリティ監査（署名検証など）を実施する。
